# AsyncLogSplitter

Инструмент для распределенной обработки системных логов, оптимизированный под `asyncio`. Проект использует `aiofiles` для предотвращения блокировки событийного цикла (Event Loop) при выполнении операций ввода-вывода.

## Использование `aiofiles` в проекте

В коде реализованы следующие возможности библиотеки:

1.  **Асинхронные контекстные менеджеры**: `async with aiofiles.open()` гарантирует корректное закрытие файлов без блокировки потока.
2.  **Асинхронная итерация**: Конструкция `async for line in f` позволяет эффективно читать файлы построчно, не загружая всё содержимое в RAM.
3.  **Управление ФС через `aiofiles.os`**: 
    * `path.exists` — неблокирующая проверка наличия директорий.
    * `mkdir` / `remove` — асинхронное создание и удаление объектов файловой системы.
    * `rename` — перенос/архивация файлов по завершении обработки.
    * `stat` — получение атрибутов файла (размер, права доступа) без остановки цикла.
4.  **Буферизация**: Использование `await f.flush()` для контроля записи данных на диск.
